# Congelamento formale — Microstep 6.2.2
## Behavioral Snapshot (Livello 1)

**Microstep:** 6.2.2 — Behavioral Snapshot (Livello 1)  
**Fase:** 6 — Behavioral Memory  
**Stato:** ❄️ FROZEN — CONGELATO  
**Tipo:** Layer derivativo tecnico (non decisionale, non semantico)

Questo documento **congela formalmente** il microstep 6.2.2. Non introduce nuovo codice né modifica implementazioni esistenti. Fissa il perimetro architetturale e le regole di non-regressione.

---

## 1. Stato del microstep

- **Microstep:** 6.2.2 — Behavioral Snapshot (Livello 1)
- **Fase:** 6 — Behavioral Memory
- **Stato:** CONGELATO
- **Tipo:** Layer derivativo tecnico (non decisionale, non semantico)
- **Visibilità UX:** Nessuna (shadow)
- **Dipendenze:**
  - Dipende **solo** da Behavioral Snapshot (Livello 0)
  - **MUST NOT** dipendere da eventi, dominio o feature
- **Dipendenze inverse:** Nessuna — nessuna feature **MUST** leggere direttamente L1

---

## 2. Dichiarazione architetturale vincolante

Il Behavioral Snapshot (Livello 1):

- è una **trasformazione numerica tecnica** dei dati L0
- **non interpreta** i dati
- **non assegna significato**
- **non prende decisioni**
- **non espone stati concettuali**
- **non abilita feature**

Ogni valore prodotto in L1 **SHALL** essere:

- numerico
- continuo o discreto
- privo di etichette semantiche
- privo di soglie decisionali

---

## 3. Responsabilità ammesse (chiuse)

Il Livello 1 **può solo**:

- normalizzare grandezze quantitative
- calcolare delta tra snapshot consecutivi
- calcolare rapporti matematici semplici
- mantenere last-known-state tecnico
- scrivere snapshot numerici in uno store dedicato

Questo elenco è **chiuso**. Qualsiasi nuova responsabilità **MUST NOT** essere aggiunta a 6.2.2; richiede un microstep successivo (es. 6.2.3 o 6.3).

---

## 4. Responsabilità vietate (vincolanti)

È **formalmente vietato** introdurre in 6.2.2:

- soglie (es. “se > N allora…”)
- stati discreti (“attivo”, “inattivo”, “freddo”, “caldo”)
- scoring interpretativi
- ranking
- priorità
- giudizi
- etichette semantiche
- mapping verso comportamenti UX
- logica decisionale di qualsiasi tipo

Qualsiasi logica di questo tipo **viola il congelamento** e **SHALL** essere implementata in blocchi successivi (es. 6.3 — Signal Derivation Layer).

---

## 5. Isolamento e disattivabilità (confermati)

Rimane vincolante che:

- il dominio **MUST NOT** leggere L1
- il dominio **MUST NOT** conoscere L1
- L1 **SHALL** essere completamente disattivabile tramite `NoOpBehavioralSnapshotL1Store` (o equivalente)
- con L1 ON o OFF:
  - output dei use case **MUST** restare invariato
  - stato di dominio **MUST** restare invariato
  - side-effect persistenti **MUST** restare invariati

---

## 6. Relazione con blocchi futuri

- **6.2.2 NON È** un Signal Layer
- **6.2.2 NON È** un Decision Layer
- **6.2.2 NON abilita** feature

Ogni:

- soglia
- classificazione
- stato (“ghosted”, “inactive”, “important”)
- decisione

**MUST** vivere **esclusivamente** in un layer successivo (es. **Blocco 6.3 — Signal Derivation Layer**), con specifica dedicata.
