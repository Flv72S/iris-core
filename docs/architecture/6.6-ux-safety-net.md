## 2. Natura del BLOCCO 6.6

Il BLOCCO 6.6 è definito come:

- **Guardrail architetturale UX**
- **Modulo di sicurezza percettiva**
- **Non-feature**
- **Non intelligente**
- **Non adattivo**
- **Non visibile come capacità**

Il suo scopo esclusivo è **impedire regressioni percettive**, ovvero impedire che il sistema:

- “parli” senza autorizzazione
- suggerisca senza richiesta
- si manifesti come intelligente o proattivo
- accumuli memoria o inferenze in contesti futili
- rompa il principio di *Silence-First*

---

## 3. Invarianti congelati (vincolanti)

Con il congelamento del BLOCCO 6.6 diventano **invarianti architetturali**:

### 3.1 Futile Chat Isolation (6.6.1)

- Le chat classificate come **futili** (brevi, emoji-only, meme, sandbox emotiva):
  - **NON** alimentano memoria comportamentale
  - **NON** alimentano AI
  - **NON** generano suggerimenti
  - **NON** partecipano a feature future
- La classificazione è **conservativa** e **fail-safe**
- **Nessuna** logica semantica o interpretativa è consentita

---

### 3.2 Silence-First Defaults (6.6.2)

- L’assenza di output è lo **stato di default**
- **Nessun** suggerimento, hint, placeholder o UI reattiva è consentito senza:
  - capability esplicita
  - tier valido
  - kill-switch disattivato
- Il silenzio è uno **stato verificabile**, non una mancanza temporanea

---

### 3.3 UX Integrity Tests (6.6.3)

- **Devono** esistere test che verificano che:
  - **nulla** “parli” senza autorizzazione
  - **nessuna** feature emerga implicitamente
  - il comportamento UX rimanga **identico** dopo refactor o nuove feature
- I test sono **di regressione percettiva**, non funzionale

---

## 4. Relazione con altri blocchi

Il BLOCCO 6.6:

- È **indipendente** dal dominio
- **Non** introduce feature
- **Non** abilita comportamenti
- **Non** legge Behavioral Memory
- **Non** modifica Capability Registry

È **concettualmente simmetrico** a:

- **6.3.3 — Kill-Switch Globale**, ma sul piano UX/percettivo

## 5. Divieti espliciti (post-congelamento)

Dopo questo congelamento è **vietato**:

- aggiungere output UX “minimi” o “di cortesia”
- introdurre suggerimenti passivi non richiesti
- usare Behavioral Snapshot in contesti futili
- bypassare Silence-First per “migliorare l’esperienza”
- reinterpretare il silenzio come fallback temporaneo

Qualsiasi violazione **richiede** **nuovo microstep formale fuori dal BLOCCO 6.6**.
