## §2 — Ambito del Semantic Engine Core (8.2.x)

Elenco esplicito di ciò che **ESISTE**:

- **SemanticEngine** — modulo centrale che accetta input read-only (Fase 7), invoca aggregazione e (opzionalmente) ordinamento e explanations; restituisce SemanticSnapshot; kill-switch globale (engine OFF → snapshot vuoto)
- **Semantic Aggregation (8.2.1)** — contratto append-only, meccanico; merge degli output dei SemanticModule senza filtri, priorità o risoluzione conflitti
- **Ranking / Priority / Weighting meccanico (8.2.2)** — ordinamento descrittivo, deterministico; rank/priority/weight come dati; nessuna selezione; kill-switch ranking (OFF → ordine 8.2.1)
- **Explanations opzionali e non autoritative (8.2.3)** — strato esplicativo in `SemanticSnapshot.explanations`; **MUST NOT** essere parte del Read Model 7.5; **MUST NOT** attribuire intent alla Fase 7; kill-switch explanations (OFF → explanations: [])
- **Kill-switch come meccanismo di degradazione** — semantic-engine, semantic-ranking, semantic-explanations; ciascuno con effetto **preciso e limitato** quando OFF (vedi §5)

---

## §3 — Ambito ESCLUSO

Elenco esplicito di ciò che **NON ESISTE** nel Semantic Engine Core 8.2.x:

- **Decisioni** — nessun meccanismo che “scelga” un esito, un’azione o un elemento
- **Policy applicative** — nessuna policy che attivi, disattivi o modifichi comportamento in base a regole semantiche
- **Enforcement** — nessun enforcement di vincoli o “raccomandazioni” vincolanti
- **View / Projection** — nessun layer di vista, proiezione o presentazione derivata; **NON ESISTE** in 8.2.x (eventuale = 8.3.x o Fase 9)
- **Presentation semantics** — nessuna semantica di “presentazione” o “vista finale”
- **UX copy o suggerimenti vincolanti** — le explanation sono opzionali e non autoritative; **MUST NOT** prescrivere o “consigliare” in modo vincolante
- **Fonte di verità diversa dalla Fase 7** — la fonte di verità per i dati **resta** la Fase 7 (Read Model, balance, limiti); lo snapshot è **overlay** reversibile

---

## §4 — Invarianti non negoziabili

Le seguenti invarianti **MUST** restare vere per tutto il ciclo di vita del Semantic Engine Core 8.2.x:

- **Aggregazione append-only** — merge senza override, senza deduplicazione semantica, senza rimozione
- **Nessuna selezione** — tutti i contributi restano presenti; nessun filter, pick, “migliore N”, cutoff
- **Nessuna risoluzione conflitti** — conflitti **NON** risolti semanticamente; nessun “vincitore” o “priorità effettiva”
- **Nessuna autorità semantica** — ranking e explanation sono **descrittivi**; **MUST NOT** determinare azioni o policy
- **Overlay reversibile** — ogni strato (ranking, explanations) **MUST** essere disattivabile; **MUST** degradare senza residui
- **Kill-switch → degradazione verso Fase 7 pura** — con engine OFF, comportamento **MUST** essere identico a assenza di Fase 8; con ranking/explanations OFF, effetto **MUST** essere limitato (vedi §5)

## §5 — Kill-switch matrix

| Component ID             | Effetto quando OFF | Limitazione |
|--------------------------|--------------------|-------------|
| **semantic-engine**      | SemanticEngine restituisce **snapshot vuoto** (createEmptySnapshot). Nessuna aggregazione, nessun modulo invocato. | Effetto: nessun overlay; sistema = Fase 7 pura. |
| **semantic-ranking**     | Nessun ordinamento applicato. Snapshot **identico** a 8.2.1 (ordine di aggregazione). states/contexts/rankings/explanations/policies **invariati** nel contenuto. | Effetto: solo ordine = append-order; nessuna vista ordinata. |
| **semantic-explanations**| Snapshot con **explanations: []**. states, contexts, rankings, policies **invariati**. | Effetto: nessuno strato esplicativo; nessun testo/reason. |

**Nessun** kill-switch modifica la Fase 7, i DTO 7.x o il Read Model 7.5.

## §7 — Anti-pattern vietati

È **VIETATO** introdurre o suggerire, nel perimetro 8.2.x o nei documenti che lo definiscono:

- **“Risultato finale”** — **MUST NOT** presentare lo snapshot o un sottoinsieme come “risultato finale” vincolante
- **“Elemento vincente”** — **MUST NOT** identificare un “vincitore”, “migliore” o “scelto”
- **“Consigliato”** (in senso prescrittivo) — **MUST NOT** usare suggestion come comando o raccomandazione vincolante
- **“Migliore opzione”** — **MUST NOT** ordinare o presentare in modo che “primo” o “massimo” sia interpretato come scelta del sistema
- **“Perché il sistema ha deciso”** — **MUST NOT** attribuire decisioni o intenzionalità alla Fase 7 o all’engine
- **Semantica UX mascherata** — **MUST NOT** introdurre copy, messaggi o narrative che prescrivano comportamento sotto la copertura di “explanation”
- **View / projection / presentation** — **MUST NOT** aggiungere viste derivate, proiezioni o layer di presentazione nel Core 8.2.x
