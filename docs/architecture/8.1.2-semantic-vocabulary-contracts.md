## §3 — Tipologie semantiche di base (primitive)

Queste sono le **SOLE** primitive semantiche ammesse.

### 3.1 — SemanticState

Stato interpretativo derivato, non persistente, overlay.

**Contratto:**

- **MUST NOT** essere uno stato tecnico
- **MUST NOT** sostituire stati 7.1.2 o 7.6
- **MUST NOT** modificare il comportamento base

**Forma:**

```
SemanticState {
  id: SemanticStateId
  scope: SemanticScope
  validity: TemporalValidity
  killSwitchable: true
}
```

**Esempi ammessi:** `active`, `inactive`, `at_risk`, `eligible`, `recommended`

**Esempi vietati:** `correct`, `optimal`, `should_do`, `best_choice`

---

### 3.2 — SemanticContext

Contesto interpretativo associato a: azione, evento, stato.

**Contratto:**

- È **descrittivo**, non prescrittivo
- **MUST NOT** introdurre obblighi
- **MUST NOT** giustificare limiti tecnici

**Forma:**

```
SemanticContext {
  id: SemanticContextId
  origin: Derived | UserDeclared
  expiresAt?: Timestamp
}
```

---

### 3.3 — SemanticRanking

Strato di ordinamento esplicito, revocabile.

**Contratto:**

- **MUST NOT** cambiare la fonte dati
- **MUST NOT** alterare l’ordine base Fase 7
- **MUST** essere sempre disattivabile

**Forma:**

```
SemanticRanking {
  criteria: DeclaredCriteria
  direction: Asc | Desc
  reversible: true
}
```

---

### 3.4 — SemanticMemory

Uso controllato di storico o tempo.

**Contratto:**

- **MUST** essere isolato dalla Fase 7
- **MUST NOT** modificare limiti 7.6
- **MUST** essere sempre invalidabile

**Forma:**

```
SemanticMemory {
  window: TimeWindow
  decay?: DecayFunction
  resettable: true
}
```

---

### 3.5 — SemanticExplanation

Strato narrativo opzionale.

**Contratto:**

- **MUST NOT** far parte del Read Model 7.5
- **MUST NOT** attribuire intenzionalità alla Fase 7
- **MUST** essere sempre removibile

**Forma:**

```
SemanticExplanation {
  message: LocalizedText
  reasonCode?: DeclaredReason
  optional: true
}
```

---

### 3.6 — SemanticPolicy (overlay di safety)

Governance semantica sopra i limiti tecnici.

**Contratto:**

- **MUST NOT** sostituire 7.6
- **MUST NOT** autorizzare ciò che 7.6 vieta
- **MUST** degradare sempre verso 7.6

**Forma:**

```
SemanticPolicy {
  appliesIf: SemanticCondition
  fallback: TechnicalLimit
}
```

## §5 — Regole di composizione

Le primitive **MUST NOT**:

- fondersi con DTO 7.x
- diventare obbligatorie

- dichiarare durata
- dichiarare kill-switch

## §6 — Kill-switch contract

Ogni elemento semantico **MUST** implementare:

- **disable(): void**

**Effetto garantito:** nessun residuo; nessuna decisione parziale; sistema = Fase 7 pura.
